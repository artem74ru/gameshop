# Настройка переменных окружения в Vercel

## Проблема

Если API не работает после деплоя на Vercel, скорее всего не настроены переменные окружения.

**Важно:** `.env` файл НЕ должен быть в Git (он уже в `.gitignore`). На Vercel переменные окружения настраиваются через веб-интерфейс.

---

## Решение: Добавить переменные окружения в Vercel

### Шаг 1: Откройте настройки проекта в Vercel

1. Перейдите на https://vercel.com/dashboard
2. Найдите ваш проект `gameshop`
3. Нажмите на него
4. Перейдите в раздел **"Settings"** (в верхнем меню)
5. В левом меню выберите **"Environment Variables"**

### Шаг 2: Добавьте переменные окружения

#### Переменная 1: RAWG_API_KEY

1. Нажмите **"Add New"** или **"Add"**
2. Заполните:
   - **Key (Name)**: `RAWG_API_KEY`
   - **Value**: ваш ключ от RAWG API (скопируйте из вашего локального `.env` файла)
   - **Environment**: выберите ВСЕ три:
     - ✅ Production
     - ✅ Preview
     - ✅ Development
3. Нажмите **"Save"**

#### Переменная 2: RAWG_BASE_URL

1. Нажмите **"Add New"** или **"Add"**
2. Заполните:
   - **Key (Name)**: `RAWG_BASE_URL`
   - **Value**: `https://api.rawg.io/api`
   - **Environment**: выберите ВСЕ три:
     - ✅ Production
     - ✅ Preview
     - ✅ Development
3. Нажмите **"Save"**

### Шаг 3: Передеплой проекта

После добавления переменных окружения нужно передеплоить проект:

1. Перейдите в раздел **"Deployments"**
2. Найдите последний деплой
3. Нажмите на три точки (⋮) справа
4. Выберите **"Redeploy"**
5. Или просто сделайте новый push в GitHub (Vercel автоматически передеплоит)

---

## Как найти ваш RAWG API ключ

Если вы забыли ваш ключ:

1. Откройте ваш локальный `.env` файл в проекте
2. Найдите строку: `RAWG_API_KEY=ваш_ключ_здесь`
3. Скопируйте значение после `=`

Или получите новый ключ:
- Перейдите на https://rawg.io/apidocs
- Зарегистрируйтесь/войдите
- Создайте новый API ключ

---

## Проверка

После передеплоя проверьте:

1. Откройте ваш сайт на Vercel
2. Перейдите в каталог (`/catalog`)
3. Должны загрузиться игры
4. Если игры загружаются - всё работает! ✅

---

## Альтернативный способ: Добавить при создании проекта

Если вы еще не задеплоили проект, можно добавить переменные окружения прямо при импорте:

1. При импорте проекта найдите раздел **"Environment Variables"**
2. Добавьте переменные там
3. Затем нажмите **"Deploy"**

---

## Важно помнить

- ✅ Переменные окружения на Vercel = переменные из `.env` локально
- ✅ `.env` файл НЕ должен быть в Git (безопасность)
- ✅ После добавления переменных нужно передеплоить проект
- ✅ Выбирайте все окружения (Production, Preview, Development) для каждой переменной
